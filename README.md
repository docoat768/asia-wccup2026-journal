# asia-wccup2026-journal

一个用于归档与发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库旨在收集、整理并托管一系列独立的 HTML 页面。这些页面可能涉及不同主题、用途或时间点，彼此之间没有强关联。仓库本身不绑定任何特定域名或网站，仅作为一个静态页面集合的存储与发布空间。

## 目录说明

```
.
├── index.html          # 仓库入口页面（如有）
├── pages/              # 存放各独立 HTML 页面的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 页面所需的资源文件（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要页面存放目录，每个 HTML 文件代表一个独立页面。
- `assets/`：页面引用的公共资源文件，按类型分目录存放。
- 根目录下的 `index.html` 为可选入口页，用于展示页面列表或导航。

## 页面归档说明

每个 HTML 页面均为独立完整的文档，可以单独打开查看。页面之间不共享数据或状态，也不依赖外部服务（除非页面自身声明）。归档遵循以下原则：

- 页面内容保持原始状态，不进行修改或重定向。
- 文件名使用有意义的英文或数字组合，避免歧义。
- 如需更新页面，会保留旧版本或另存为新文件，确保历史可追溯。

## 使用方式

你可以通过以下方式访问本仓库中的页面：

1. **直接浏览**：克隆仓库后，在本地用浏览器打开任意 HTML 文件。
2. **GitHub Pages**：如果仓库开启了 GitHub Pages 功能，可以通过对应 URL 访问（请参考仓库设置中的 Pages 配置）。
3. **其他托管**：将仓库内容部署到任何支持静态文件的服务器。
